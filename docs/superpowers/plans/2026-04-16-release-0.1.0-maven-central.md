# bluetape4k-graph 0.1.0 Maven Central 배포 계획 (Git Flow)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Git Flow 방식으로 release/0.1.0 브랜치를 생성하고, 버전을 업데이트한 뒤 Maven Central에 배포한다.

**Architecture:** develop → release/0.1.0 (버전 범프 + 검증) → main (태그 v0.1.0) → develop (다음 스냅샷 버전 복귀). 배포는 `main` 브랜치에서 NMCP aggregation 플러그인을 통해 Central Portal에 직접 업로드한다.

**Tech Stack:** Kotlin 2.3 + Gradle 8.x + NMCP (nmcp_aggregation) + Maven Central Portal API

| 다음 개발 버전 | `0.2.0-SNAPSHOT` |

## Task 1: release 브랜치 생성 및 버전 업데이트

**Files:**
- Modify: `gradle.properties`
- Modify: `CHANGELOG.md`

- [ ] **Step 1: release 브랜치 생성**

```bash
git checkout develop
git pull origin develop
git checkout -b release/0.1.0
```

Expected: `Switched to a new branch 'release/0.1.0'`

- [ ] **Step 2: gradle.properties 버전 업데이트**

`gradle.properties` 파일에서 다음 두 줄을 수정한다:

```
# 변경 전
baseVersion=0.0.1
snapshotVersion=-SNAPSHOT

# 변경 후
baseVersion=0.1.0
snapshotVersion=
```

Edit 도구로 수정:
- `baseVersion=0.0.1` → `baseVersion=0.1.0`
- `snapshotVersion=-SNAPSHOT` → `snapshotVersion=`

- [ ] **Step 3: 버전 변경 확인**

```bash
grep -E "baseVersion|snapshotVersion" gradle.properties
```

Expected 출력:
```
baseVersion=0.1.0
snapshotVersion=
```

- [ ] **Step 4: CHANGELOG.md 업데이트**

`[Unreleased] - 0.0.1-SNAPSHOT` 섹션을 `[0.1.0] - 2026-04-16`으로 변경하고, 상단에 새 `[Unreleased]` 섹션을 추가한다.

```markdown

### Added

### Changed

### Fixed

### Added
...기존 내용...
```

```bash

```

## 문제 해결 가이드

| 문제 | 원인 | 해결 |
|------|------|------|
| `publishAggregationToCentralPortal` 인증 오류 | `~/.gradle/gradle.properties`에 `centralUser`/`centralPassword` 미설정 | Central Portal에서 토큰 발급 후 `~/.gradle/gradle.properties`에 `centralUser=<token>`, `centralPassword=<secret>` 추가 |
| 빌드 실패 (`BUILD FAILED`) | 컴파일 오류 또는 의존성 문제 | `./gradlew build -x test --stacktrace`로 상세 오류 확인 |
| `merge --no-ff` 충돌 | develop와 main 간 충돌 | `git status`로 충돌 파일 확인 후 수동 해결, `git add`, `git merge --continue` |
| Testcontainers 오류 | Docker 미실행 | Docker Desktop 실행 후 재시도, 또는 TinkerPop 테스트만 실행 |
| Central Portal 반영 지연 | Maven Central 인덱싱 시간 | 10~30분 대기 후 https://central.sonatype.com 재확인 |

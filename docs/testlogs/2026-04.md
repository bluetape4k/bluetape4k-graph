### 코드 리뷰에서 발견된 이슈 (tests 로 고정만 하고 수정은 별도 PR 권장)

1. **`GraphProperties` vs `AgePropertySerializer` 중복** — `graph-core/.../utils/GraphProperties.kt` 와 `graph-age/.../sql/AgePropertySerializer.kt` 가 거의 동일한 로직. 후자가 `LocalDate`/`LocalDateTime`/`Map` 처리를 추가로 가지고 있어 사실상 graph-core 쪽이 dead code이며 기능적으로도 stale. 통합 권장.
2. **Cypher 문자열 escape 부족** — `"'"` 만 escape 하고 `\`(백슬래시), `\n`, `\r` 등은 그대로 문자열에 포함됨. 테스트에서는 현재 동작을 그대로 고정했으므로, 후속으로 AGE/Neo4j 파서 호환성 확인 필요.
3. **`else` 브랜치의 잠재적 Cypher 인젝션** — `toString()` + 단일 escape 로만 처리. 스키마 타입 외의 객체가 들어올 경우 위험. 스키마 타입 미지원 시 예외를 던지는 방향을 검토.
4. **`serializableUID` 오타** — 모든 모델 클래스가 `const val serializableUID` 를 선언하지만 Java 직렬화 규약상 이름은 `serialVersionUID` 이어야 해서 현재는 단순 상수. 직렬화 호환성을 원한다면 companion object가 아닌 클래스 본체에 `private const val serialVersionUID` 로 두어야 함.
5. **`EdgeLabel` DSL 불일치** — `VertexLabel` 은 `stringList`, `json` 을 제공하지만 `EdgeLabel` 은 없음. 의도가 있다면 주석, 아니면 동기화 필요.
